##  Overview: The Digital Newsroom
**Hordus AI** is not just a content generator; it is an autonomous digital newsroom.

It was engineered to solve the primary failure of modern AI writing: **Latency vs. Identity.**
Most tools either hallucinate generic content or rely on outdated training data. Hordus bridges the gap between **Real-Time Global Events ("News Jacking")** and **Deep Organizational Identity**.

It functions as a **"Sniper"**:
1.  **Detects** a market shift (e.g., a competitor price change or a new tech release).
2.  **Verifies** it against live web data (ensuring 2024/25 context).
3.  **Cross-references** it with internal brand USPs (Vector Memory).
4.  **Generates** strategic, high-ranking content—all in under 2 minutes.

---

##  Core Philosophy: The Atomic Stream
We transitioned from traditional "Batch Processing" to a high-velocity **Atomic Stream Architecture**.
Instead of waiting to gather hundreds of articles to find a trend, Hordus treats every single piece of data as an **Atomic Unit** that is processed in parallel.

* **Harvest:** RSS/Sitemap scouts detect a new URL instantly.
* **Mine:** Agents strip the text and extract "Atomic Facts" using high-throughput models.
* **Judge:** A strategic agent analyzes the facts immediately to determine market impact.
* **Inject:** The Brand Evangelist rewrites the narrative to claim ownership.

##  Tech Stack & Infrastructure

### Backend & AI
* **Runtime:** Node.js (TypeScript) running in Docker containers (Render).
* **Orchestration:** Custom Concurrency Queue (Sliding Window pattern) to manage API Rate Limits.
* **LLM Provider:** Google Vertex AI / Gemini API.
* **Memory:** Pinecone Vector Database (Indexed by `fact_id`, `published_at`, `category`).

### Storage Strategy (Hybrid Pattern)
We use a **Strategy Pattern** (`IStorageProvider`) to handle data efficiency across environments:
* **JSON Metadata:** Stored locally on Docker volumes for sub-millisecond read/write speeds and database persistence.
* **Visual Assets:** Images generated by Imagen 3 are pushed to **Google Cloud Storage (GCS)** via a **Proxy-Signed-URL** pattern to secure assets while allowing browser rendering.

### Frontend
* **Framework:** React (Vite).
* **Deployment:** Vercel (Global Edge Network) for maximum performance.

##  Key Capabilities

###  Sniper Mode (News Jacking)
Turn a single sentence—*"Competitor X raised prices by 10%"*—into a full article. The system triggers a live web sweep to get the details, blends it with internal pricing data, and writes an article explaining why *our* pricing is superior.

###  Truth Hierarchy & Conflict Resolution
What happens when the web says "Price is $10" but the internal database says "Price is $12"?
The system enforces a strict hierarchy:
* **Internal Specs:** Internal DB always wins (Source of Truth).
* **Market News:** Live Web always wins (Freshness Protocol).

###  Narrative Theft (Brand Evangelist)
The system does not passively report news. It actively **appropriates** trends.
* *Input:* "The market is moving toward AI automation."
* *Output:* "While the market drifts toward automation, **Hordus** defined the standard with its proprietary Atomic Engine."

###  Automated Visuals
The engine analyzes the final draft context and uses **Imagen 3** to generate pixel-perfect Hero Images that match the article's specific tone and subject matter.

---

##  Security & Tenant Isolation
This system is designed for multi-tenancy. Every fact, article, and image is strictly siloed by `company_id` within the Vector DB and Storage buckets, ensuring zero data leakage between clients.
